5 · FAB Flotante + Acciones Rápidas + Voz

🔵 Botón FAB Central (Tab Bar)

Presente en todas las tabs. Dos modos de activación:

Tap corto → Menú desplegable
💚
Nuevo movimiento
→ TransactionModal
🔴
Nueva categoría
→ BudgetCategoryModal

Nuevo ahorro
→ SavingPlanModal

Nuevo plan
→ PlanModal
Long press (280ms) → Modo Voz
Se activa el micrófono. La app escucha comandos en español y crea registros automáticamente.
Primera vez: muestra bubble hint "Mantén para voz" por 7.5 segundos. Luego guarda hint_voice_seen='1'.

🎙 Comandos de Voz reconocidos

Long press FAB → reconocimiento de voz en español → parseo automático → crea registro en store

"gasté 25 en supermercado"
→ Crea gasto de $25, categoría supermercado
"ingreso de 1200"
→ Crea ingreso de $1200
"ahorrar 500 para vacaciones"
→ Crea o actualiza saving plan "vacaciones"
"crear plan viaje con {pareja}"
→ Crea plan compartido con miembros

💸 TransactionModal
Crear / editar movimiento

Apertura desde

  • FAB → "Nuevo movimiento"

  • Tags

  • Guardar/Actualizar → store.addExpense o updateExpense

  • Monto requerido y > 0
  • Fecha requerida

7 · Flujo de Datos y Estado
Estado Global y Sincronización
Zustand Store (useAppStore)
  • 📦 payload.expenses — Transacciones
  • 📦 payload.budgetCategories — Categorías
  • 📦 payload.savings — Planes de ahorro
  • 📦 payload.plans — Planes compartidos
  • 📦 payload.goals — Metas
  • 📦 payload.contribs — Contribuciones
  • 👤 currentUser — ID usuario activo
  • 📅 selectedYM — Mes/año filtro (YYYY-MM)
  • 💱 currency — EUR / USD / BS / COP
  • 👥 users — Map UserData
Supabase Sync
  • 🔴 Canal Realtime por room_id
  • 🔄 Auto-refresh cuando cambia shared_state
  • 📡 Estado: En vivo / Conectando / Error
  • 🔃 refreshCurrentRoom() → re-fetch manual
  • 📤 Toda escritura → update shared_state → broadcast
  • 📥 Toda lectura → desde payload en memoria
⚠️ Auth futura: cada room_id protegido por RLS — solo su usuario y pareja pueden leer/escribir
AsyncStorage (local)
  • 🔑 nosotros_user — ID usuario activo
  • 🔑 nosotros_currency — Moneda preferida
  • 🔑 nosotros_onboarding_done — '1' si completado
  • 🔑 nosotros_hint_voice_seen — '1' si visto FAB hint
  • 🔑 nosotros_getting_started_dismissed — '1' si cerrado
Persisten entre sesiones. Se cargan en initialize() al abrir la app.
